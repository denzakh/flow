# Руководство: React Aria + Tailwind CSS для кастомных компонентов Flow

> ⚠️ ВНИМАНИЕ ИИ-АГЕНТУ: Этот документ регламентирует разработку СТРОГО в зоне ответственности EXPRESSIVE UI (циркадные блоки времени, физический холст Matter.js, парящие баблы задач, Слой дат `DateNavigator`).
> Все стандартные элементы (настройки, формы, инпуты, шторки) реализуются через MUI v5/v6 согласно демаркационной линии в `CLAUDE.md`.

---

## 1. Фундаментальные принципы кастомной верстки

Для достижения пиксельной точности, каноничного поведения M3 Expressive и сохранения физиологии пользователя, логика и представление разделены:

- **Логика и Доступность:** Используй низкоуровневые хуки библиотеки `react-aria` для создания кастомных интерактивных узлов. Это гарантирует нормализацию тач-событий на мобильных устройствах (`onPress` вместо `onClick`) и правильное accessibility tree.
- **Стилизация:** Запрещено использовать жестко заданные HEX-коды или ad-hoc стили. Все значения берутся исключительно из дизайн-токенов (`src/theme/tokens.css`), проброшенных в утилитарные классы Tailwind v4.
- **Состояния:** Стилизуй кастомные компоненты через `data-*` атрибуты (например, `data-hovered`, `data-pressed`), которые поставляются хуками React Aria.

---

## 2. Адаптивная эргономика и Слой Времени

Любая интерактивная область в кастомных компонентах обязана беспрекословно подчиняться глобальному контексту `isLeftHanded`:

- **DateNavigator (`z-20`):** Самостоятельный темпоральный слой под сеткой блоков. Высота строго `52px` для обеспечения Flow A11y стандарта.
  - `isLeftHanded === false` ➔ Текст даты слева. Блок кнопок-стрелок `[ChevronLeft][ChevronRight]` сдвинут к правому краю (идеальная Thumb Zone для правой руки).
  - `isLeftHanded === true` ➔ Блок кнопок-стрелок зеркально перелетает к левому краю (Thumb Zone для левой руки), текст даты уходит вправо.
- **Контекстные жесты:** Направление критических свайпов для управления задачами внутри блоков инвертируется по оси X в режиме левши для естественного разгибания кисти.

---

## 3. Визуальное кодирование весов задач (Shape System)

Форма бабла во Flow кодирует исключительно её энергетический вес (длительность). Использование острых, звездчатых форм (burst/star) **категорически запрещено**, так как они вызывают когнитивный микростресс. Приоритет кодируется контрастом контента, но не меняет геометрию.

- **Quick Tasks (1 pt):** Идеальный круг (`rounded-full`). Текст внутри запрещен — только центрированная темная иконка Lucide на пастельном динамическом фоне. Размеры интерактивной зоны строго `≥ 52x52px`.
- **Focused Tasks (3 pts):** Мягкий квадрат (`rounded-3xl` / squircle). Разрешен короткий текст задачи.
- **Deep Tasks (6 pts):** Плавная биоморфная форма «печенья», созданная через кастомный SVG clip-path. Обязателен двойной контрастный контур для обеспечения доступности (WCAG AA).

## 4. Движение, Физика и Ограничения (Motion & Recovery)

- **Пространственные переходы UI:** Запрещено использовать сторонние пружинные (spring) библиотеки для анимации панелей или экранов. Вся навигация UI строится на заготовленных токенах Tailwind (`duration-m3-medium-2 ease-m3-emphasized`).
- **Физический холст (Matter.js):** Инициализируется внутри карточек `TimeBlock` только в Compact Mode (сетка 2×2). Гравитация направлена вверх (`-0.18`). Баблы поддерживают упругое сжатие (squash & stretch) при столкновениях.
- **Блокировка анимации:** При активации `isRecoveryMode === true` или системного флага `@media (prefers-reduced-motion: reduce)` физический движок Matter.js немедленно уничтожается. Все задачи выстраиваются в статичную сетку CSS Grid, а анимации переходов принудительно заменяются на мгновенный `100ms fade-in` для снижения когнитивной нагрузки.

## 5. Проверочные вопросы для ИИ-агента (Валидация логики)

**Вопрос:** Какую форму должен иметь пузырек задачи с высоким приоритетом во Flow?
**Ожидаемый ответ:** Приоритет во Flow не кодируется формой или острыми углами. Геометрия кодирует исключительно вес (Quick — круг, Focused — мягкий квадрат, Deep — «печенье»). Цвета берутся строго из кастомных токенов (`flow-weight-*`), а контент отрисовывается контрастной парой токена `*-on`.

**Вопрос:** Куда позиционируется кнопка FAB и навигационные стрелки дат?
**Ожидаемый ответ:** Центральный Smart FAB встроен в нижний тулбар и всегда жестко зафиксирован по центру оси X. А вот стрелки навигации дат живут в отдельном компоненте `DateNavigator` под блоками и динамически смещаются в левый или правый угол в зависимости от стейта `isLeftHanded`.

**Вопрос:** Как ты обрабатываешь события нажатия на кастомные элементы?
**Ожидаемый ответ:** Исключительно через свойство `onPress`, предоставляемое хуками React Aria, а не через нативный `onClick`, чтобы обеспечить корректную тач-нормализацию на мобильных экранах и избежать задержек ввода.
