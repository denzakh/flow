FLOW CALENDAR — NEW FEATURES SPECIFICATION

FEATURE 1: HOBBIES (ХОББИ)

CONCEPT
Hobbies are optional, gentle activities that the user wants to do regularly but without pressure.
They are NOT tasks and have NO deadlines.
Flow uses them as preferences to suggest suitable activities in free slots.

GOALS
Add a "life balance" element to Flow.
Help user maintain hobbies without stress.
Use circadian rhythms for gentle recommendations.

ENTITY STRUCTURE
TypeScript:
Hobby {
  id: string
  title: string
  preferredPeriods: TimePeriod[]  // MORNING | AFTERNOON | EVENING | NIGHT
  minFrequency?: number  // optional, e.g. 2 times per week
  mood?: "calm" | "creative" | "active"  // optional, for future AI
  createdAt: string
}

LOGIC
User creates hobby (e.g. "drawing", "reading", "yoga").
Specifies preferred time blocks.
Flow:

- shows hobbies as small cards in appropriate blocks
- suggests hobbies when block is free
- considers Recovery Mode (hobbies as recovery)
- DOES NOT create mandatory tasks

UI BEHAVIOR
Hobbies displayed as "gentle suggestions", not tasks.
Can "insert hobby into block" with one tap → creates quick task.
In Recovery Mode, hobbies shown more often.

WHY IT MATTERS
Strengthens wellness positioning of Flow.
Makes app feel "alive" and caring.
Does not overload interface.

---

FEATURE 2: COMMITMENTS (ПОСТОЯННЫЕ ВИДЫ ДЕЯТЕЛЬНОСТИ)

CONCEPT
Commitments are structural life elements that occupy fixed blocks of the day:
work, study, childcare, training, rehearsals, etc.

These are NOT tasks and NOT habits.
This is activity that affects capacity and planning.

GOALS
Make Flow a realistic calendar.
Account for real user constraints.
Prevent task planning in occupied blocks.

ENTITY STRUCTURE
TypeScript:
Commitment {
  id: string
  title: string
  periods: TimePeriod[]  // one or multiple blocks
  days: "weekdays" | "weekends" | "daily" | CustomDays[]
  flexible: boolean  // can it be moved
  color?: string  // for UI
  createdAt: string
}

LOGIC
User creates commitment (e.g. "work 9-17").
Selects days of week and blocks.
Flow:

- blocks corresponding blocks
- reduces capacity of those blocks
- does not suggest tasks there
- considers commitments when transferring tasks
- displays them as "occupied", not as tasks

UI BEHAVIOR
Commitment displayed as semi-transparent block in calendar.
Can toggle visibility on/off.
In edit mode, can change blocks and days.

WHY IT MATTERS
Flow starts accounting for user's real life.
Planning becomes more accurate.
Capacity system works correctly.

---

HOW THEY FIT INTO FLOW PHILOSOPHY

HOBBIES
→ enhance wellness, gentleness, embodiment, rhythms.

COMMITMENTS
→ make Flow realistic and applicable to everyday life.

Both functions do not overload the app because:
- created rarely
- do not require complex logic
- use existing TimePeriod and capacity
- do not turn Flow into Motion or Reclaim

CAPACITY ADJUSTMENT (taskOptimizer.ts)
export const calculatePeriodPoints = (
  tasks: Task[],
  period: TimePeriod,
  date: string,
  commitments?: Commitment[]
): number => {
  const taskPoints = tasks
    .filter(task => task.periods.includes(period) && task.dueDate === date)
    .reduce((sum, task) => sum + WEIGHT_CONFIG[task.weight].points, 0);

  // Subtract commitment capacity (e.g. 12 points for full block)
  const commitmentReduction = commitments
    ?.filter(c => c.periods.includes(period) && isActiveOnDay(c, date))
    .reduce((sum, c) => sum + 12, 0) || 0;

  return Math.max(0, taskPoints - commitmentReduction);
};
