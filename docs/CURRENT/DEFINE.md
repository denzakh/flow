## 📊 Краткое резюме исследования

Проведён количественно-качественный опрос (n=99) о привычках планирования. Респонденты — студенты, наёмные сотрудники, фрилансеры/предприниматели. Выявлены 6 ключевых кластеров болей, построены 3 персоны и сформулированы HMW-вопросы для стадии Ideate.

**Ключевой вывод:** Пользователи хотят не просто список задач, а **инструмент, учитывающий их энергию и состояние**, с минимальным трением ввода и визуальной поддержкой приоритетов.

## 1. 🧩 Аффинная диаграмма

Открытые ответы (n=44 про фрустрации, n=23 про "magic feature", n=47 про recovery) сгруппированы в 6 кластеров.

### C1 · Фрагментация и хаос инструментов
**Инсайт:** Пользователи используют 2–4 инструмента одновременно, ни один не даёт целостной картины дня.

| Цитата | Связь с Flow |
|---|---|
| "Что они все в разных инструментах и не пересекаются" | Единый день-вид с 4 блоками (Утро/День/Вечер/Ночь) |
| "На бумажке удобно, но нельзя всё таскать с собой" | Мобильное приложение + голосовой ввод |
| "Sometimes I find some To-Do's from apps I haven't used in years" | Централизованное хранение + автоочистка |
| "It is not organized — listed/written separately" | Единый интерфейс вместо разрозненных списков |
| "Слишком сложный интерфейс, много кликов чтобы добавить задачу" | Голосовой ввод + минимум действий |

### C2 · Уведомления не работают
**Инсайт:** Напоминания либо не приходят, либо игнорируются — пользователи забывают заглядывать в план.

| Цитата | Связь с Flow |
|---|---|
| "Я забываю в них смотреть" | Fade-in будильник + Recovery Banner |
| "Не всегда вижу напоминания" | Активные триггеры, не пассивные нотификации |
| "A veces Google Calendar falla en mandarme notificaciones" | Надёжная система напоминаний |
| "I would like a notification to help me remember to look at it" | Умные напоминания о просмотре плана |

### C3 · Нет учёта энергии и состояния
**Инсайт:** Инструменты игнорируют физическое и эмоциональное состояние — план одинаков в понедельник и в пятницу вечером.

| Цитата | Связь с Flow |
|---|---|
| "Organizarme sola las tareas para distribuir la energía" | Циркадные блоки + capacity system (12 pts/блок) |
| "Вносить данные голосом, встроенный ИИ как личный помощник" | Голосовое управление на 3 языках |
| "Автоматическое перерасписание задач с учётом моих приоритетов" | adjustTaskPeriods() с учётом ёмкости |
| "Если нет ресурса — на задачи уходит больше времени и происходит выгорание" | Recovery Mode + гибкие блоки |
| "I see if I can move tasks to another moment — my mental health is important" | Анти-выгорание как базовая фича |

### C4 · Задачи накапливаются и не переносятся
**Инсайт:** Ручной перенос — самая частая боль. Незавершённые задачи вызывают тревогу или просто теряются.

| Цитата | Связь с Flow |
|---|---|
| "Yes, I move them to a different day when I have more energy" | Автоперенос с учётом состояния |
| "Переношу не важные задачи на потом" | Автоматическая приоритизация |
| "They pile up until I clear them out" | preventTaskAccumulation() |
| "It just snowballs without actually fixing the stress problem" | Recovery Mode вместо накопления |
| "Хвалить меня за выполненные задачи" | Gamification + позитивное подкрепление |

### C5 · Перегрузка и визуальный стресс
**Инсайт:** Длинный список задач вызывает тревогу сам по себе — визуальная плотность работает против пользователя.

| Цитата | Связь с Flow |
|---|---|
| "The list feels like it's mocking me when I can't bring myself to do them" | Пузырьковая визуализация (не список) |
| "Its sometimes overwhelming to look at" | Лимит 12 pts/блок — нет перегрузки |
| "Overload of features/info" | Минималистичный M3-дизайн |
| "STUFF DOES GET LOST — lack of overview" | Чёткая структура 4 блоков |
| "Nothing is clearly categorised" | Визуальное разделение Quick/Focused/Deep |

### C6 · Приоритизация не автоматическая
**Инсайт:** Пользователи хотят помощи в определении важного — текущие инструменты не помогают расставить приоритеты.

| Цитата | Связь с Flow |
|---|---|
| "Identify what work is more important than another" | Веса задач: Quick (1pt) / Focused (3pts) / Deep (6pts) |
| "Хотелось бы инструменты для расставления приоритетности задач" | AI-подсказка веса при голосовом вводе |
| "Мне важна визуальная составляющая — чтобы были выделены приоритетные задачи" | Цветовое кодирование + размер пузырей |
| "Try to focus on 3 MOST important things for the day" | Топ-3 задачи дня с акцентом |
| "Compruebo que mi rendimiento es menor — дать себе время" | Автоматическая корректировка при усталости |

## 3. 👥 Персоны пользователей

### Персона 1 · Маша, 24 · Студентка · Группа A
**Quote:** "Я всё успею, но потом. А потом — снова потом."

**Контекст:** Учится на дневном, параллельно подрабатывает. Планирует в голове или в мессенджере себе. Знает что утром работается лучше, но редко использует это — соцсети сильнее.

| Цели | Боли | Поведение |
|---|---|---|
| Не забывать важные дедлайны | Список задач пугает — смотрит и не знает с чего начать | Планирует оптимистично — выполняет реалистично |
| Не чувствовать вину к вечеру | Мелкие дела съедают день | Переносит задачи вручную каждый день |
| Тратить меньше времени на «что делать дальше» | Забывает проверять план | Снижает нагрузку через откладывание |

**Flow для Маши:** Quick/Focused разделение + авто-перенос + голосовой ввод

### Персона 2 · Лора, 32 · Наёмный сотрудник · Группа A
**Quote:** "К вечеру чувствую вину что не сделала важное, хотя день был полным."

**Контекст:** Работает в офисе или удалённо. Использует цифровой календарь, иногда блокнот. Осознаёт свои пики энергии, но план не подстраивает — рабочие встречи диктуют расписание.

| Цели | Боли | Поведение |
|---|---|---|
| Защитить утреннее время для глубокой работы | Жёсткая сетка встреч vs гибкость задач | Пересматривает приоритеты в конце дня |
| Понять почему день «не так пошёл» | Все инструменты разрозненны | Осознанно снижает нагрузку при усталости |
| Меньше переносить задачи вручную | Сложно оценить ёмкость дня | Хочет голосовой ввод на ходу |

**Flow для Лоры:** Блоки дня + capacity system + Recovery Mode

### Персона 3 · Алекс, 29 · Фрилансер · Группа B
**Quote:** "Без внешней структуры мой день — это просто список тревог."

**Контекст:** Фрилансер с несколькими проектами. Нет фиксированного расписания — сам создаёт структуру. Высокий риск выгорания. Знает что энергия циклична, планирует хаотично.

| Цели | Боли | Поведение |
|---|---|---|
| Не работать вечером когда нет ресурса | «Токсичный» тайм-блокинг — жёсткий план → вина | Снижает нагрузку через делегирование или разбивку |
| Видеть реальную нагрузку дня | Задачи накапливаются пока не становятся критическими | Работает лучше вечером или по состоянию |
| Не забывать свои проекты ради клиентских | Нет системы приоритетов между своим и клиентским | Ищет gamification и вознаграждение |

**Flow для Алекса:** Recovery Mode + гибкие блоки + Idea Bank

## 4. 🎯 Problem Statements

**Формула:** [Пользователь] — [потребность] — потому что [инсайт].

### Персона 1 · Маша (Студентка)
**Маша** — студентка, которой нужно **понимать, что делать прямо сейчас и не чувствовать вины за неоконченное**, — потому что её **список задач вызывает тревогу, а не помогает сосредоточиться**.
Маша is a/an 24-year-old student who plans in her head or in messenger and procrastinates on important tasks
who needs to understand what to do right now and not feel guilty about unfinished tasks
because her task list causes anxiety and paralysis rather than helping her focus — she feels guilty by evening knowing she did small tasks but left the important ones undone.

### Персона 2 · Лора (Наёмный сотрудник)
**Лора** — наёмный сотрудник, которой нужен **инструмент, учитывающий её энергию и автоматически расставляющий задачи**, — потому что **к вечеру важное остаётся несделанным, хотя день был насыщенным**.
Лора is a/an 32-year-old office employee who uses digital calendar but struggles with fragmented tools and manual rescheduling
who needs an intelligent planning tool that automatically protects time for deep work based on her energy peaks and task priorities
because by evening she feels guilty that important tasks remain undone even though her day was full of meetings and small tasks — she wants to understand "why the day didn't go as planned".

### Персона 3 · Алекс (Фрилансер)
**Алекс** — фрилансер, которому нужна **гибкая структура дня без жёсткого тайм-блокинга**, — потому что **детальный план при усталости вызывает вину, а не помогает восстановиться**.
Алекс is a/an 29-year-old freelancer who creates his own daily structure and experiences burnout from rigid planning
who needs a flexible energy-based planning system that adapts to his state and automatically reschedules unfinished tasks without judgment
because detailed time-blocking causes guilt when he's tired or plans change, and tasks accumulate until they become critical — he needs structure without toxicity.

## 6. 🗓️ Customer Journey Map · "Утро Алекса"

| Этап | Действие | Мысль | Эмоция | Точка контакта | Возможность Flow |
|---|---|---|---|---|---|
| **1. Пробуждение** | Открывает глаза, берёт телефон | "Что там на сегодня? Надеюсь, не много" | 😐 Нейтрально → 😟 Лёгкая тревога | Lock screen | Fade-in вид дня с ёмкостью (не список!) |
| **2. Оценка состояния** | Вспоминает что вчера устал | "Сегодня буду работать полегче" | 🤔 Задумчивость | — | Recovery Mode предлагает снизить ёмкость |
| **3. Добавление задачи** | Вспомнил идею на ходу | "Надо записать, пока не забыл" | 😤 Фрустрация от ввода руками | Голосовой ассистент | "Flow, добавь позвонить клиенту — Deep" |
| **4. Начало работы** | Смотрит план | "С чего начать? Всё важное..." | 😰 Перегрузка | Главный экран | Топ-3 задачи дня + пузыри по размеру |
| **5. Выполнение** | Делает первую задачу | "Так, это заняло больше времени" | 😓 Усталость | Таймер задачи | Авто-корректировка оставшейся ёмкости |
| **6. Срыв плана** | Пришло срочное от клиента | "Всё, день пошёл не по плану" | 😠 Раздражение → 😞 Вина | Уведомление | Автоперенос + "Это нормально, перестроим" |
| **7. Вечерний обзор** | Смотрит что сделал | "Не всё, но главное — сделал" | 😌 Облегчение + 😊 Гордость | Evening Summary | "Ты сделал 8/12 pts — отлично!" |

## 7. 📈 Ключевые количественные находки

| Находка | Цифра | Что означает для Flow |
|---|---|---|
| Переносят задачи вручную или дают накапливаться | **~56%** | Автоперенос — не nice-to-have, а базовая потребность |
| Хотят чтобы инструмент учитывал состояние | **60% (4-5/5)** | Самый высокий Ликерт — circadian-подход востребован |
| Трудно оценить ёмкость дня | **43% (4-5/5)** | Capacity system с визуальным лимитом нужна |
| Утро = пик концентрации | **36%** | Равно вечеру (36%) — блоки важны, утро не единственный выбор |
| Важная задача не сделана из-за мелких | **«Иногда» у 54% гр.А** | Система весов Quick/Focused/Deep решает эту боль |
| Уровень вины одинаков в обеих группах | **~15% в А и Б** | Recovery Mode нужен всем, не только фрилансерам |
| Жёсткий план → вина у фрилансеров | **50% при тайм-блоках** | Гибкие блоки важнее точного расписания |

## 8. 📊 Success Metrics

| Метрика | Цель (3 мес) | Как измеряем |
|---|---|---|
| **Day Completion Rate** | ≥70% дней с ≥60% выполнением | % дней с закрытыми задачами |
| **Voice Input Coverage** | ≥40% задач добавлено голосом | Доля голосовых задач от всех |
| **Recovery Mode Usage** | ≥25% пользователей активировали хотя бы раз | % пользователей с Recovery-сессиями |
| **Manual Reschedule Rate** | ≤10% (было ~56%) | Автоперенос vs ручной |
| **7-Day Retention** | ≥50% | Пользователи, вернувшиеся на 7-й день |
| **NPS** | ≥40 | Опрос "порекомендуете ли Flow" |
