# Flow State Management & Interaction Rules

## 1. Состояния Интерактивности (M3 Interaction States via React Aria)
Любой интерактивный элемент (кнопка, карточка, инпут) управляется через состояния React Aria и визуализируется через State Layers:
- **Enabled (Дефолт):** Базовый цвет из `theme.json`.
- **Hovered:** Использовать хук `isHovered` из React Aria. Накладывается оверлей `on-surface` или `primary` с прозрачностью 8% (`opacity-[0.08]`).
- **Focused:** Использовать `isFocused`. Добавляется четкий контур (outline) толщиной 2px с отступом, цвет `md.sys.color.focus-outline`.
- **Pressed:** Использовать `isPressed`. Оверлей прозрачностью 10% (`opacity-[0.10]`) + мгновенная анимация сжатия (scale-98).
- **Disabled:** Элемент теряет контраст, `opacity-38` согласно гайдлайнам M3, событие клика блокируется на уровне архитектуры.

## 2. Глобальное Состояние Приложения (Global State Architecture)
Архитектура данных Flow делится на три уровня:
1. **User Bio-Resource State (Состояние ресурсов пользователя):**
   - Текущий уровень энергии (шкала от 0 до 100).
   - Динамика изменения энергии (растет/падает).
   - Зависит от времени суток и выполненных задач.
2. **Task State (Состояние задач):**
   - Списки задач, их статус (backlog, in-progress, done).
   - "Вес" задачи (сколько био-ресурса она требует: легкая, средняя, тяжелая).
3. **Timer/Focus State (Состояние фокуса):**
   - Активен ли режим глубокого фокуса.
   - Оставшееся время таймера.

## 3. Доступность Состояний (Accessibility & ARIA)
- Изменение любого состояния (например, задача отмечена как выполненная) должно мгновенно транслироваться в accessibility tree.
- Использовать `aria-selected`, `aria-checked`, `aria-expanded` строго в соответствии с поведением компонентов из React Aria.
- При изменении глобального состояния (например, падение уровня энергии ниже критического) использовать `aria-live="polite"`, чтобы уведомить пользователя через скринридер, не прерывая его текущее действие.
