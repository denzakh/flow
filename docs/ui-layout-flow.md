# Архитектура, компоновка и иерархия интерфейса Flow (Layout Constitution)

> ⚠️ ИНСТРУКЦИЯ ДЛЯ ИИ: Этот документ определяет пространственную структуру и правила наложения слоев (Z-Index) для гибридной архитектуры (MUI + React Aria + Tailwind v4). 
> Любое изменение компоновки должно строго соответствовать макету "Gallery-Mobile 2.png".

---

## 1. Иерархия контейнеров и Слои интерфейса

Каждый элемент интерфейса должен находиться на своем строго определенном уровне. Перекрытие или смешивание слоев без учета Z-Index ЗАПРЕЩЕНО.


### Глобальная иерархия слоев (Снизу вверх / по оси Z):

1. **Background Layer (`z-0`):** Глобальный фоновый градиент приложения и декоративные размытые пастельные пятна (циркадный фон).
2. **Temporal Layer / Слой Времени (`z-10`):** Компонент `DateNavigator` (текст даты и стрелки переключения дней), расположенный строго под сеткой блоков и над нижним тулбаром.
3. **Expressive Canvas Layer (`z-20`):** - Сетка 4 квадрантов `Day View` (`MorningBlock`, `AfternoonBlock`, etc.). Главный рабочий холст.
   - Физические холсты Matter.js внутри блоков в Compact режиме.
4. **Toolbar Layer / Пульт управления (`z-30`):** Компонент `FlowToolbar` (монолитная панель во всю ширину экрана с центральным Smart FAB).
5. **Overlay Layer / Системный интерфейс (`z-50`):** Выезжающие шторки (MUI `SwipeableDrawer`), модальные окна настроек (MUI `Dialog`), уведомления об ошибках ввода и перегрузке емкости (`Snackbar`).

## 2. Спецификации адаптивности и Блокировки Layout Shift

Для предотвращения дергания интерфейса (Layout Shift) при загрузке данных, смене состояний или отсутствии задач, геометрия холста жестко зафиксирована:

### Режимы отображения Day View:
- **Compact Mode (Сетка 2×2):** Высота каждого блока жестко зафиксирована (`min-height: 253px` на мобильных устройствах). Внутри работает физика Matter.js. Пузыри не могут вылетать за пределы родительского квадранта (`z-10`).
- **Expanded Mode (Раскрытый блок):** Активируется по клику на блок или через кнопку переключения видов. Контейнер блока занимает 100% ширины, высота становится `auto` (растет по содержимому). Физика Matter.js уничтожается, задачи рендерятся в виде вертикального списка стандартных MUI карточек.
- **Vertical Stack Mode (Хронологический список):** Все 4 блока идут друг под другом. Поведение аналогично Expanded Mode.

## 3. Инженерный контракт для isLeftHanded (Биомеханика)

При изменении глобального стейта `isLeftHanded === true` ИИ-агент обязан перестроить лейаут Слоя Времени и Тулбара по зеркальной схеме:

- **DateNavigator (`z-20`):**
  - `false` ➔ Текст даты слева, блок кнопок `[ChevronLeft][ChevronRight]` справа (в Thumb Zone правой руки).
  - `true` ➔ Блок кнопок `[ChevronLeft][ChevronRight]` слева (в Thumb Zone левой руки), текст даты уходит вправо.
- **FlowToolbar (`z-30`):**
  - Центральный Smart FAB всегда неподвижен (строго по центру оси X).
  - `false` ➔ `[InboxIcon]` слева ➔ `[Central FAB]` ➔ `[SettingsIcon]` справа.
  - `true` ➔ `[SettingsIcon]` слева ➔ `[Central FAB]` ➔ `[InboxIcon]` справа.

## 4. Правила Валидации для ИИ-агента (Чек-лист перед коммитом)

Перед генерацией кода или изменением верстки любого экрана ИИ-агент обязан проверить:

1. **Не ломает ли элемент Z-иерархию?** (Физический пузырь задачи не должен всплывать над тулбаром или модалкой).
2. **Зарезервировано ли место под ошибки?** (В формах ввода под управлением MUI TextField всегда должно быть зафиксировано пространство под `supporting text`, чтобы появление ошибки не сдвигало нижние элементы).
3. **Как интерфейс ведет себя в Recovery Mode?** (При `isRecoveryMode === true` или `prefers-reduced-motion` контейнеры Day View немедленно отключают Matter.js, переводя отображение задач в статичную сетку CSS Grid, а все анимации переходов сжимаются до `100ms fade-in`).
4. **Учтены ли размеры кликабельных зон?** (Все интерактивные элементы в `DateNavigator` и `FlowToolbar` должны иметь физическую зону тапа `≥ 52x52px`).
