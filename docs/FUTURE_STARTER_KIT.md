# Манифест: Среда сверхбыстрого прототипирования (Indie-Hacker Starter Kit)

Цель: Создать изолированный, переиспользуемый каркас (Boilerplate) для мгновенной проверки продуктовых гипотез в коде силами одного дизайнера и ИИ-агентов, сокращая время от идеи до живого UI до нескольких часов.

## Архитектурная матрица (Три слоя истины)
### Слой 1: Динамический мост (Design Tokens)
Исключить ручной перенос визуальных параметров (HEX, шрифты, отступы). Архитектура должна быть полностью управляемой извне.

**Источник**: Figma Variables / Tokens Studio.

**Транслятор**: Файл theme.json в корне слоя стилей.

**Сборщик**: Автоматический маппинг JSON-токенов в конфигурацию tailwind.config.ts с семантическими ролями Material Design 3 (bg-md-sys-color-primary, text-md-sys-color-on-primary).

**Результат**: Смена бренда, темы или геометрии всего приложения происходит заменой одного JSON-файла, без прикосновения к коду компонентов.

### Слой 2: Анатомия интерфейса (Headless UI)
Исключить повторное написание логики доступности, фокуса и интерактивности.

**База**: React 19 + Нестилизованные примитивы React Aria / React Stately.

**Состав заготовки**: В репозитории изначально лежат «слепые», но стопроцентно рабочие компоненты: Button.tsx, Modal.tsx, Select.tsx, Popover.tsx, Calendar.tsx.

**Логика внутри**: В них намертво зашита обработка кликов, тапов, жестов, навигация с клавиатуры и фокус-лейеры (State Layers для Hover/Focus/Press).

**Результат**: ИИ или человек управляют только внешним видом через утилитарные классы Tailwind. Компоненты технически не могут сломаться или вести себя некорректно.

### Слой 3: Контекстный мозг для ИИ (AI Context Management)
Среда проектируется с расчетом на то, что 80% рутинного кода будут писать Cursor и Cline. Их действия должны быть жестко ограничены системой пре-промптов.

**Инструменты контроля**: Преднастроенные файлы .cursorrules и .clinerules в корне, требующие от моделей соблюдения чистоты слоев.

**База знаний**: Папка docs/templates/ с пустыми бланками ui-rules.md (геометрия, сетка 4px, правила M3) и state-management.md (обработка состояний данных).

**Результат**: При создании нового проекта достаточно заполнить текстовые спецификации в docs/, и ИИ-агент мгновенно понимает контекст, генерируя код без галлюцинаций и ошибок в путях импорта.
